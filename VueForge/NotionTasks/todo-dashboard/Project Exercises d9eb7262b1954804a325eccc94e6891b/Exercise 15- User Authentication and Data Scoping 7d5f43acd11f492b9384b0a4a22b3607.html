Exercise 15- User Authentication and Data Scoping

  1. Install @8base/auth@^2.6.6 package and
  1. Create a new subscriber role in the 8base dashboard under AppServices -> Roles
  1. Create a new authentication profile in 8base dashboard under AppServices->Authentication with the following values:
    1. Name: App Auth
    1. Type: 8base Authentication
    1. Self Signup: Open to All
    1. Roles: Subscriber
  1. After saving the authentication profile, add the following values to your .env file from the new screen
VITE_AUTH_DOMAIN=
VITE_AUTH_CLIENT_ID=
VITE_AUTH_PROFILE_ID=
  1. Setup 8base auth client using the @8base/auth package
  1. Create an AuthUserStore to expose methods for interacting with auth client and to store reactive user data
  1. Create a login route that calls the login method from the AuthUserStore
  1. Create a logout route that calls the logout method from the AuthUserStore
  1. Create an auth/callback route to create the user in 8base if they don’t already exist and set the access token to state and local storage
  1. Add authorization header to apollo client
  1. Change the permissions on Guest Role to be custom filters (filters available in resources below)
  1. Attach user’s team to the board when creating
