Exercise 0 - Before you get started

Vue.js Forge is fast approaching and I hope you’re as excited as we are to get started!

To prepare for the build of our Trello clone SaaS application, there are a couple of things you should take care of beforehand.

On the conference day, you will be coding live so we recommend that you complete these 4 items ASAP so you have time to troubleshoot if anything goes wrong. The full setup should take between 20 to 30 minutes.

Later this week we will connect you with your new teammates and you will have the chance to discuss any issues you faced.

1) Download the Project Boilerplate Code

In order to eliminate some of the bootstrapping work that comes with the project, we’ve provided some boilerplate code for you to download.

➡️ Download the boilerplate code from Github now and run npm install to download dependencies from NPM.

2) Get Setup with 8base

We’ll be using 8base as a powerful backend to provide data storage with a GraphQL API, server side functions, and authentication.

➡️ Go ahead and create an 8base account to save time during the event. We also ask that you create a new workspace on the free tier. You can call it whatever you’d like 🙂. I opted for “Trello Clone”.

3) Grab your Kendo UI Free Trial

Save time building the project by using out of the box components provided by Kendo UI. During the build, we’ll make use of the super slick drawer component for creating the main navigation of the app, as well as many of the other components for common needs like buttons, modal overlays (dialogs), and more!

➡️ Start your free trial of Kendo UI today (no CC required) and follow these instructions to add the license key to the boilerplate code.

Also feel free to browse all their available components to see what all is possible.

4) Get Started with TwicPics

Images typically account for the majority of the weight on any web page or app. Serving the right size and quality image at the right time can drastically improve performance. We’ll incorporate TwicPics in the project to take care of this for us.

➡️ Create a TwicPics account, create a workspace, and create a domain with the name of your choice. We’ll be using that to domain to connect TwicPics to the project.

5) Sign-Up for ButterCMS

Finally, during the build, we’ll also create a marketing-style landing page, an essential for any SaaS app. In order to make the page manageable by a non-technical marketing team, we’ll utilize ButterCMS and hook it up to a Nuxt static site.

Also take note of the Read API Token on the settings page within ButterCMS. We will be using this to connect ButterCMS to the landing page.

6) Join your team 🥳

If you haven't already, join the Discord server, and then look out for your ticket code in your email to join your teammates.

During the event, you will have access to your team to chat, get help, support others, and have fun coding together. You can decide if you prefer to solve each exercise by yourself or delegate tasks between the team and commit your code to a single repository.

A Little Prep Goes a Long Way

Thanks so much for taking care of this beforehand! Completing these steps now will ensure that you can get to coding as quickly as possible and optimize the valuable time you have together with your team.

After you’ve completed these steps, be on the lookout for an email soon where you’ll be connected with your team!
