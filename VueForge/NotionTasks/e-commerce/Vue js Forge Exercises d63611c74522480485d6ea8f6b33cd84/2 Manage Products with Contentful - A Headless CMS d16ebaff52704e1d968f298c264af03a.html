2. Manage Products with Contentful - A Headless CMS (Brittany Walker)

Goal

Fetch product data from a headless CMS (in this case Contentful) instead of from a hard-coded JSON file so that products are easier to manage.

Prerequisites

  • Start on the branch 2-start
  • Contentful account created
  • Stripe account created
  • See Preparation Guide for more details

Steps

  1. Create a new content type in Contentful called “Product”
  1. Give the product the following fields:
    • Name: Short Text
    • Summary: Long Text
    • Price: Decimal Number (Help text: price in cents - this is necessary for Stripe)
    • Description: Long Text
    • Image: Media, many files
  1. Create some of your own products to sell. Feel free to use hot-sauce like we did or come up with your own offerings 😃
  1. Consume the products in the Nuxt application. You can do this with the official Contentful JavaScript SDK (it’s already installed in the project and is provided to the Nuxt app as $contentful via a plugin)
  1. Setup a webhook so that every time a product is created in Contentful it will also be created in Stripe with the same id
  • Screenshots and Webhook Payload
    {
  "id": "{ /payload/sys/id }",
  "name": "{ /payload/fields/name/en-US }",
  "description": "{ /payload/fields/description/en-US }",
  "default_price_data": {
    "currency": "USD",
    "unit_amount_decimal": "{ /payload/fields/price/en-US }"
  }
}
  1. By the end of the exercise the products on the homepage should no longer be powered by the static JSON data but should be coming fully from Contentful

Also, the products created in Contentful should be created with the same id in Stripe

Bonus #1

  1. Also fetch the individual products for the dedicated product page
  1. By the end of the bonus exercise the data for the single product page should be coming completely from Contentful and no longer rely on the static JSON data
