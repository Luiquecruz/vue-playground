8. Use BigCommerce For Advanced Functionality and Quick Start Times (Erik Christensen)

Goal

At this point we’ve done a mighty fine job creating a simple DIY e-commerce application and really focused on learning more about Vue.js and Nuxt. But what if we could get a storefront solution that’s a lot more robust with a fraction of the effort.

Enter Big Commerce! It can act as a headless e-commerce platform with all the bells and whistles (returns, shipping, tax, etc) and can be quickly setup to work with an official Big Commerce Nuxt.js front-end template.

Prerequisites

  • Big Commerce account created
  • This exercise does NOT start from a branch in the boilerplate (instead we start from the bc-nuxt-vue-starter)

Steps

  1. Clone the official Big Commerce Nuxt starter template
git clone git@github.com:bigcommerce/bc-nuxt-vue-starter.git
  1. Change directories to the project and install dependencies 
cd bc-nuxt-vue-starter
npm install
  1. Open up the project in your code editor. If you’re using VS Code you can run the following command
code .
  1. Rename .env.example to .env. Next we’ll want to start filling these in.
  1. We can find these in the Big commerce Dashboard under Settings > API Accounts
  1. Click “Create API Account”
  1. Keep token type as “V2/V3 API token”. Call your API account “Nuxt App”. And set all the permissions to the highest permission (most will be “Modify”)
  1. Also grab the hash after /stores in the API path and store it as the value in .env for STORE_HASH (in the screenshot that would be: 1n0nn82q99)
  1. When you hit save, you’ll be presented with a pop-up that includes your api ids and secrets.
  1. Add these for the following .env variables in the project
API_TOKEN
API_CLIENT_ID
API_SECRET
  1. Next you’ll need to create a storefront channel you can do that with the following command now that those API variables are in place.
npm run create-storefront-channel --name [whatever-name-you-want]
  1. The output from the above command will look something like this:
{
  data: {
    id: 1189415,
    name: 'nuxt-site',
    platform: 'custom',
    type: 'storefront',
    date_created: '2022-08-24T17:56:39Z',
    date_modified: '2022-08-24T17:56:39Z',
    external_id: '',
    icon_url: 'https://s3.amazonaws.com/bc-channel-platform/channel-icons/custom.svg',
    is_listable_from_ui: false,
    is_enabled: true,
    is_visible: true,
    status: 'active'
  },
  meta: {}
}
  1. Copy the output id as you’ll need that to run the next command to create a storefront token 
npm run create-storefront-token --channelId [the-id-from-above] --hostURL http://localhost:3000
